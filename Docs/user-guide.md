# LootMaster 2.0 — Руководство пользователя

## Первый запуск

**Шаг 1 — Открыть базу данных**
Нажми **"Открыть SQLite базу"** на панели инструментов. Выбери файл `*.sqlite3` / `*.db` с данными ArcheAge (имена предметов, NPC, категории).

**Шаг 2 — Открыть JSON файлы лута**
Нажми **"Открыть JSON дропа"**. Можно выбрать несколько файлов сразу (например `loot1.json`, `loot_doodad.json`). Ожидаемый формат:
```json
[ { "npc_id": 123, "items": [ { "item_id": 456 }, ... ] }, ... ]
```
После загрузки в нижней части экрана появится статистика: сколько предметов и NPC загружено.

> Следующие запуски — сессия восстанавливается автоматически. Повторно выбирать файлы не нужно.

## Интерфейс

```
┌─ Тулбар: кнопки + поиск + фильтры ──────────────────────────────┐
├─ Синие плашки с именами загруженных JSON файлов ─────────────────┤
│                                                                   │
│  ◄ ТАБЛИЦА ПРЕДМЕТОВ (60%) ►  │  ◄ ПРАВАЯ ПАНЕЛЬ (40%) ►        │
│                                │                                  │
│  Все предметы с данными дропа  │  • Инфо о выбранном предмете    │
│  Клик по строке = выбор        │  • Настройки предмета           │
│                                │  • Настройки категории          │
│                                │  • Навигация                    │
│                                │  • Браузер NPC                  │
│                                │  • Сводка (статистика)          │
└────────────────────────────────┴──────────────────────────────────┘
│ Статус-бар (текущее состояние / спиннер загрузки)                 │
```

**Цвета строк в таблице:**
- **Зелёный** — предмету назначена группа напрямую (item-level)
- **Жёлтый** — группа унаследована от категории (category-level)
- **Белый** — не обработан

---

## Основная работа — назначение лут-групп

### Предмет

1. Кликни на предмет в таблице слева
2. В правой панели — секция **"Настройки предмета"**
3. Введи **Группу** (число) и/или **Шанс** (дробное число, разделитель `,` или `.`)
4. Нажми **"Применить"** — строка станет зелёной
5. Для сброса — **"Сбросить"**

### Категория (пакетное назначение)

1. Выбери любой предмет из нужной категории
2. В секции **"Настройки категории"** введи группу/шанс
3. Нажми **"Применить ко всей категории"** — все предметы этой категории окрасятся жёлтым
4. Если у отдельного предмета уже есть своя настройка — она имеет **приоритет** над категорией

> **Итоговая логика:** `item group > category group`, `item chance > category chance`

## Фильтры и поиск

| Элемент | Действие |
|---|---|
| Поле **"Поиск:"** | Фильтрует по ID, имени предмета, ID и имени категории, ID и именам NPC |
| Чекбокс **"Только необработанные"** | Скрывает уже назначенные предметы |
| Чекбокс **"Подсвечивать категории"** | Включает/выключает жёлтую подсветку |
| Чекбоксы **"Кол. NPC ID"** / **"NPC имена"** | Показывает/скрывает соответствующие колонки |

## Навигация

| Кнопка | Горячая клавиша | Действие |
|---|---|---|
| ← Предыдущий | — | Предыдущий предмет в отфильтрованном списке |
| Следующий → | — | Следующий предмет |
| Следующий необработ. | — | Перепрыгнуть к ближайшему необработанному |
| Сохранить прогресс | **Ctrl+S** | Явное сохранение |

---

## Браузер NPC

Секция **"NPC выбранного предмета"** в правой панели:

- Показывает список NPC, с которых падает выбранный предмет
- При клике на NPC — в нижней таблице появляются все предметы этого NPC с их текущими группами/шансами
- **Двойной клик** по предмету в нижней таблице — переход к этому предмету в главной таблице

---

## Сохранение и экспорт

| Действие | Результат |
|---|---|
| **Ctrl+S** / "Сохранить прогресс" | Сохраняет рабочий файл `%APPDATA%/LootMaster/loot_group_progress.json` |
| Авто-сохранение | Происходит автоматически после каждого "Применить" / "Сбросить" |
| **"Экспорт результата"** | Сохраняет итоговый JSON: `{ "items": [ { "item_id": X, "chance": Y }, ... ] }` |

## Формат файлов

### Рабочий файл прогресса (`%APPDATA%/LootMaster/loot_group_progress.json`)
```json
{
  "db_path": "/path/to/compact.server.table.sqlite3",
  "source_json_paths": ["/path/to/loot1.json", "/path/to/loot_doodad.json"],
  "items": {
    "12345": { "group": 1, "chance": 0.5 }
  },
  "categories": {
    "100": { "group": 2, "chance": 1.0 }
  }
}
```

### Входной JSON лута
```json
[
  {
    "npc_id": 123,
    "items": [
      { "item_id": 456 },
      { "item_id": 789 }
    ]
  }
]
```

### Экспортный JSON
```json
{
  "items": [
    { "item_id": 456, "chance": 0.5 },
    { "item_id": 789, "chance": 1.0 }
  ]
}
```

## Типичный рабочий процесс

```
1. Открыть БД + JSON (один раз)
2. Включить "Только необработанные"
3. По очереди обрабатывать предметы:
   - Если вся категория одинакова → "Применить ко всей категории"
   - Если предмет особенный → "Применить" только к нему
4. Используй браузер NPC чтобы видеть контекст дропа
5. Ctrl+S периодически
6. Когда всё готово → "Экспорт результата"
```
