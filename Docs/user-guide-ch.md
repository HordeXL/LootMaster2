# LootMaster 2.0 — 用户指南

## 首次启动

**步骤 1 — 打开数据库**

支持两个独立的数据库：

| 按钮 | 数据库 | 包含内容 |
|---|---|---|
| **"打开数据库"** | `compact.sqlite3` | items, npcs, item_categories, localized_texts |
| **"打开战利品数据库"** | `compact.server.table.sqlite3` | loots, loot_groups, loot_pack_dropping_npcs |

如果未选择战利品数据库，则会在主数据库中查找战利品表（向后兼容）。
两个路径都会保存在进度文件中，并自动恢复。

**步骤 2 — 打开掉落 JSON 文件**
点击**"打开掉落JSON"**。可以同时选择多个文件（例如 `loot1.json`、`loot_doodad.json`）。

支持两种格式：

NPC 战利品：
```json
[ { "npc_id": 123, "items": [ { "item_id": 456 }, ... ] }, ... ]
```
Doodad 战利品（世界对象）：
```json
[ { "doodad_id": 322, "loot_pack_id": 6414, "items": [ { "item_id": 456, "name": "...", "count": 1 }, ... ] }, ... ]
```
两种格式可以同时加载。doodad 文件中的 `loot_pack_id` 会被直接使用。
加载完成后，状态栏会显示统计信息：已加载多少物品、分类和 NPC。

> 后续启动时——会话会自动恢复，无需重新选择文件。
> 如果某个 JSON 文件被移动或删除——它会自动从会话中排除，不会报错。
> 不是 JSON 数组的文件（例如进度文件）会被跳过并显示警告。

**物品表格列：**

| 列 | 描述 |
|---|---|
| Item ID | 物品 ID |
| 物品 | 本地化名称 |
| Cat ID / 分类 | 分类 ID 和名称 |
| NPC ID / NPC | NPC ID 和名称 (默认隐藏，通过复选框启用) |
| 组 (物品) / 几率 (物品) | 手动设置的物品级值 |
| 组 (分类) / 几率 (分类) | 从分类继承的值 |
| 最终组 / 最终几率 | 最终值 (物品优先级高于分类) |
| Loot Pack ID | 与物品 NPC 关联的战利品包 ID |
| 组 (数据库) / 几率 (数据库) | 已写入数据库 `loots` 表的值 |
| 最小数量 / 最大数量 | 手动设置的掉落数量范围 |
| 等级 ID | 分配的物等级 ID (有效值: 物品级 → 数据库回退) |
| 总是掉落 | "必然掉落" 标志 (有效值: 物品级 → 数据库回退) |

**行颜色：**
- **绿色** — 物品直接设置了组 (物品级)
- **黄色** — 组从分类继承 (分类级)，启用分类高亮时
- **白色** — 未处理

选中的行始终显示为蓝色——即使焦点转移到右侧面板的输入字段。

> **列位置、面板宽度和窗口大小**会自动保存，下次启动时恢复 (`Data\column-settings.json`)。

## 核心工作流 — 分配战利品组

### 物品

1. 点击左侧表格中的物品
2. 在右侧面板——**"物品设置"**区域
3. 字段会自动预填：首先从保存的进度中读取，然后从数据库值中读取，否则使用默认值
4. 可用字段：
   - **组** 和 **几率** — 核心战利品参数
   - **最小数量** / **最大数量** — 掉落数量范围
   - **等级 ID** — 物品等级 ID
   - **总是掉落** — 无条件掉落标志
5. 输入所需值并点击**"应用"**—行将变为绿色
6. 要重置物品的所有字段——**"重置"**

**"应用数据库值到所有"** — 批量操作：将组、几率、等级 ID 和总是掉落从数据库设置为物品级值，适用于所有有数据库数据但未手动设置物品级组的物品。

**"撤销数据库应用"** — 撤销"应用数据库值到所有"：重置所有物品级组与组 (数据库) 匹配的物品的组和几率。

### 分类 (批量分配)

1. 从目标分类中选择任意物品
2. 在**"分类设置"**区域，输入组、几率、等级 ID 和总是掉落
3. 点击**"应用到整个分类"**—该分类的所有物品将变为黄色；等级 ID 和总是掉落直接在物品级应用
4. 要重置分类——**"重置"** (同时清除该分类所有物品的等级和总是掉落)
5. 如果单个物品已有自己的设置，则**优先于**分类设置

> **优先级规则：** `物品组 > 分类组`，`物品几率 > 分类几率`

### 战利品组的工作原理

在"物品设置"标题栏中有一个**"?"**按钮——在程序内打开帮助窗口。

| 组 | 行为 |
|---|---|
| **0** | 每个物品按其几率**独立**检查。多个物品可能同时掉落。 |
| **1, 2, 3...** | "战利品袋"：首先对照 `loot_groups.drop_rate` 掷骰——是否开启袋子。如果是，从组中选择**一件**物品 (按 `loots.drop_rate` 加权)。 |

**数据库实例 (pack_id=9327)：**
- `loot_groups`：group_no=1, drop_rate=41 666 → **0.42%** 开启袋子的几率
- `loots`：group=1 中有 952 件物品，各有不同权重
- 结果：击杀 NPC 时——首先 0.42% 几率开启袋子，然后从 952 件物品中掉落恰好**一件**

**一个 pack_id 有多个组** — 每组独立检查。每个触发的组掉落一件物品。

**公式：** `drop_rate ÷ 10 000 000 = 几率`。示例：500 000 → 5%

## 过滤和搜索

| 元素 | 操作 |
|---|---|
| **"搜索:"** 字段 | 按 ID、物品名称、分类 ID/名称、NPC ID/名称过滤 |
| **"仅未处理"** 复选框 | 隐藏已处理的物品 (绿色和黄色行) |
| **"高亮分类"** 复选框 | 切换从分类继承组的行的黄色高亮 |
| **"显示NPC ID"** / **"显示NPC名称"** 复选框 | 显示/隐藏对应的表格列 |
| **"RU"** / **"EN"** / **"ZH"** 按钮 | 切换整个界面语言。设置在会话间保存。 |

## 导航

| 按钮 | 快捷键 | 操作 |
|---|---|---|
| **← 上一个** | — | 过滤列表中的上一个物品 |
| **下一个 →** | — | 下一个物品 |

## NPC 浏览器

右侧面板中的**"所选物品的NPC"**区域：

- **NPC 列表** — 掉落所选物品的所有 NPC。标题括号中显示数量。点击 NPC 会更新下方表格。
- **"所选NPC的所有物品"** 表格 — 所选 NPC 的所有物品，带有当前组/几率和 Loot Pack ID。标题括号中显示行数。
  - **单击** — 焦点转移到主表格
  - **双击** — 跳转到主表格中的该物品 (过滤器自动清除)

## 保存和导出

| 操作 | 结果 |
|---|---|
| **Ctrl+S** / "保存进度" | 将工作文件保存到 exe 旁边的 `Data\` 文件夹 |
| 自动保存 | 每次"应用"/"重置"后自动进行 |
| **"导出结果"** | 保存最终 JSON：`{ "items": [ { "item_id": X, "chance": Y }, ... ] }` |
| **"写入数据库"** | 将所有分配的值直接写入 SQLite：upsert `loots` 和 `loot_groups` 表。写入前显示预览 (多少行将被更新/插入)。 |
| **"导入SQL到数据库"** | 将 SQL 文件直接执行到数据库。可选择多个文件。支持标准 SQL 和 Navicat 导出 (INSERT 不带列名)。新行插入，现有行 (按 `id`) 更新而不删除。 |
| **"从数据库生成补丁"** | 从选定的 SQLite 数据库读取 `loot_actability_groups`、`loot_groups`、`loot_pack_dropping_npcs`、`loots` 表，并在 `Data\` 文件夹中生成 `INSERT OR REPLACE` SQL 补丁。显示按表分组的行数预览和文件路径。**目标数据库不会被修改**——补丁需通过"导入SQL到数据库"手动应用。 |

### 导入统计

**"摘要"**区域并排显示两种导入类型的累计统计：

```
── SQL 导入 ──          ── 补丁生成 ──
已插入:  N              已插入:  N
已更新:  M              已更新:  M
总计:    N+M            总计:    N+M
```

统计在会话间累积并保存在进度文件中。

## 文件位置

所有数据存储在**可执行文件旁边**的 `Data\` 文件夹中：

```
LootMaster.exe
Data\
  loot_group_progress.json      ← 工作进度文件 (包含导入统计)
  column-settings.json          ← 列布局、面板宽度、窗口大小、语言
  patch_YYYYMMDD_HHMMSS.sql     ← 通过"从数据库生成补丁"生成的补丁文件
```

## 文件格式

### 工作进度文件 (`Data\loot_group_progress.json`)
```json
{
  "db_path": "C:/path/to/compact.sqlite3",
  "loot_db_path": "C:/path/to/compact.server.table.sqlite3",
  "source_json_paths": ["C:/path/to/loot1.json", "C:/path/to/loot_doodad.json"],
  "items": {
    "12345": { "group": 1, "chance": 0.5, "min_amount": 1, "max_amount": 3, "grade_id": 2, "always_drop": false }
  },
  "categories": {
    "100": { "group": 2, "chance": 1.0, "grade_id": 1, "always_drop": false }
  },
  "sql_imports": { "patch.sql": { "inserted": 100, "updated": 50 } },
  "db_imports":  { "source.sqlite3": { "inserted": 4109, "updated": 0 } }
}
```

### 输入战利品 JSON (NPC)
```json
[
  {
    "npc_id": 123,
    "items": [
      { "item_id": 456 },
      { "item_id": 789 }
    ]
  }
]
```

### 输入战利品 JSON (Doodad)
```json
[
  {
    "doodad_id": 322,
    "loot_pack_id": 6414,
    "items": [
      { "item_id": 456, "name": "Some item", "count": 1 }
    ]
  }
]
```

### 导出 JSON
```json
{
  "items": [
    { "item_id": 456, "chance": 0.5 },
    { "item_id": 789, "chance": 1.0 }
  ]
}
```

## 典型工作流程

```
1. 打开数据库 + JSON (一次即可，下次启动自动恢复)
2. 启用"仅未处理"
3. 逐个处理物品：
   - 如果整个分类都一样 → "应用到整个分类"
   - 如果物品特殊 → 只对它"应用"
4. 使用 NPC 浏览器查看掉落上下文
   - "组 (数据库)" 和 "几率 (数据库)" 列显示数据库中已有的值
   - Loot Pack ID 帮助识别物品属于哪个战利品池
5. 定期按 Ctrl+S 保存
6. 完成后：
   - "写入数据库" — 直接写入 SQLite (upsert loots + loot_groups)
   - 或"导出结果" — 保存 JSON 供后续处理
   - 或"导入SQL" — 将 SQL 转储直接加载到战利品数据库
   - 或"从数据库生成补丁" — 从另一个 SQLite 数据库生成补丁，然后通过"导入SQL"应用
```
