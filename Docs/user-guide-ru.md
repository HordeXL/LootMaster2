# LootMaster 2.0 — Руководство пользователя

## Первый запуск

**Шаг 1 — Открыть базы данных**

Поддерживаются две раздельные базы:

| Кнопка | База | Содержит |
|---|---|---|
| **"Открыть SQLite базу"** | `compact.sqlite3` | items, npcs, item_categories, localized_texts |
| **"Открыть БД лута"** | `compact.server.table.sqlite3` | loots, loot_groups, loot_pack_dropping_npcs |

Если БД лута не выбрана — лут-таблицы ищутся в основной базе (обратная совместимость).
Оба пути сохраняются в файле прогресса и восстанавливаются автоматически.

**Шаг 2 — Открыть JSON файлы лута**
Нажми **"Открыть JSON дропа"**. Можно выбрать несколько файлов сразу (например `loot1.json`, `loot_doodad.json`).

Поддерживаются два формата:

NPC-лут:
```json
[ { "npc_id": 123, "items": [ { "item_id": 456 }, ... ] }, ... ]
```
Doodad-лут (объекты мира):
```json
[ { "doodad_id": 322, "loot_pack_id": 6414, "items": [ { "item_id": 456, "name": "...", "count": 1 }, ... ] }, ... ]
```
Оба формата можно загружать одновременно. `loot_pack_id` из doodad-файла используется напрямую.
После загрузки в строке статуса появится статистика: сколько предметов, категорий и NPC загружено.

> Следующие запуски — сессия восстанавливается автоматически. Повторно выбирать файлы не нужно.  
> Если какой-либо JSON файл был перемещён или удалён — он автоматически исключается из сессии без ошибок.  
> Файлы, которые не являются массивом JSON (например файл прогресса), пропускаются с предупреждением.

```
┌─ Тулбар ─────────────────────────────────────────────────────────┐
│ [Открыть SQLite базу] [Открыть БД лута] | [Открыть JSON дропа]  │
│ | [Сохранить] [Экспорт] | [Записать в БД] | [Импорт SQL]        │
│ | [Поиск...] [Только необработанные] [Подсвечивать категории]    │
│ [Кол. NPC ID] [Кол. NPC имена] | [RU/EN/ZH]                      │
├──────────────────────────────────────────────────────────────────┤
├─ Синие плашки с именами загруженных JSON файлов ─────────────────┤
│                                                                   │
│  ◄ ТАБЛИЦА ПРЕДМЕТОВ (60%) ►  │  ◄ ПРАВАЯ ПАНЕЛЬ (40%) ►        │
│                                │                                  │
│  Все предметы с данными дропа  │  • Выбранный предмет (инфо)     │
│  Клик по строке = выбор        │  • Настройки предмета           │
│                                │  • Настройки категории          │
│                                │  • Навигация                    │
│                                │  • NPC выбранного предмета      │
│                                │  • Сводка (статистика)          │
└────────────────────────────────┴──────────────────────────────────┘
│ Статус-бар (текущее состояние / спиннер загрузки)                 │
```

**Колонки таблицы предметов:**

| Колонка | Описание |
|---|---|
| Item ID | ID предмета |
| Предмет | Локализованное название |
| Cat ID / Категория | ID и название категории |
| NPC ID / NPC | ID и имена NPC (скрыты по умолчанию, включаются чекбоксами) |
| Группа (предмет) / Шанс (предмет) | Назначенные вручную item-level значения |
| Группа (категория) / Шанс (категория) | Значения, унаследованные от категории |
| Итог. группа / Итог. шанс | Финальные значения (item имеет приоритет над category) |
| Loot Pack ID | ID лут-пака, привязанного к NPC предмета |
| Группа (БД) / Шанс (БД) | Значения, уже записанные в таблицу `loots` базы данных |
| Мин. кол. / Макс. кол. | Назначенные вручную минимальное/максимальное количество при выпадении |
| Грейд ID | Назначенный ID грейда предмета (эффективное: item-level → БД) |
| Всегда | Флаг "всегда выпадает" (эффективное: item-level → БД) |

**Цвета строк в таблице:**
- **Зелёный** — предмету назначена группа напрямую (item-level)
- **Жёлтый** — группа унаследована от категории (category-level), при включённой подсветке категорий
- **Белый** — не обработан

Выделенная строка всегда отображается синим — даже когда фокус переходит на поля ввода в правой панели.

> **Расположение столбцов, ширина панелей и размер окна** запоминаются автоматически между запусками (файл `Data\column-settings.json`).

## Основная работа — назначение лут-групп

### Предмет

1. Кликни на предмет в таблице слева
2. В правой панели — секция **"Настройки предмета"**
3. Поля заполняются автоматически: сначала из сохранённого прогресса, затем из значений БД, иначе значения по умолчанию
4. Доступные поля:
   - **Группа** и **Шанс** — основные параметры лута
   - **Мин. количество** / **Макс. количество** — диапазон количества при выпадении
   - **Грейд ID** — ID грейда предмета
   - **Всегда выпадает** — флаг безусловного выпадения
5. Введи нужные значения и нажми **"Применить"** — строка станет зелёной
6. Для сброса всех полей предмета — **"Сбросить"**

**"Применить из БД ко всем"** — пакетное применение: устанавливает Группу, Шанс, Грейд и Always из БД как item-level значения для всех предметов, у которых данные из БД известны, но item-level группа ещё не задана вручную.

**"Отменить из БД"** — отменяет действие "Применить из БД ко всем": сбрасывает группу и шанс у всех предметов, у которых item-level группа совпадает с Группой (БД).

### Категория (пакетное назначение)

1. Выбери любой предмет из нужной категории
2. В секции **"Настройки категории"** введи группу, шанс, грейд ID и флаг "Всегда выпадает"
3. Нажми **"Применить ко всей категории"** — все предметы этой категории окрасятся жёлтым; Грейд ID и Always применяются напрямую на item-level
4. Для сброса категории — **"Сбросить"** (также очищает Грейд и Always у всех предметов категории)
5. Если у отдельного предмета уже есть своя настройка — она имеет **приоритет** над категорией

> **Итоговая логика:** `item group > category group`, `item chance > category chance`

### Как работают группы лута

В заголовке секции "Настройки предмета" есть кнопка **"?"** — открывает справочное окно прямо в программе.

| Группа | Поведение |
|---|---|
| **0** | Каждый предмет проверяется **независимо** по своему шансу. Могут выпасть несколько предметов одновременно. |
| **1, 2, 3...** | "Мешок лута": сначала бросок против `loot_groups.drop_rate` — открывается ли мешок. Если да — выбирается **один** предмет из группы (взвешенно по `loots.drop_rate`). |

**Реальный пример из БД (pack_id=9327):**
- `loot_groups`: group_no=1, drop_rate=41 666 → **0.42%** шанс открыть мешок
- `loots`: 952 предмета в group=1 с разными весами
- Итог: при убийстве NPC — сначала 0.42% на открытие, затем из 952 предметов выпадает ровно **один**

**Несколько групп у одного pack_id** — каждая группа проверяется независимо. Из каждой сыгравшей группы выпадает по одному предмету.

**Формула:** `drop_rate ÷ 10 000 000 = шанс`. Пример: 500 000 → 5%

## Фильтры и поиск

| Элемент | Действие |
|---|---|
| Поле **"Поиск:"** | Фильтрует по ID, имени предмета, ID и имени категории, ID и именам NPC |
| Чекбокс **"Только необработанные"** | Скрывает уже назначенные предметы (зелёные и жёлтые) |
| Чекбокс **"Подсвечивать категории"** | Включает/выключает жёлтую подсветку строк, у которых группа унаследована от категории |
| Чекбоксы **"Кол. NPC ID"** / **"Кол. NPC имена"** | Показывает/скрывает соответствующие колонки таблицы |
| Кнопка **"RU"** / **"EN"** / **"ZH"** | Переключает язык всего интерфейса. Настройка сохраняется между запусками. |

## Навигация

| Кнопка | Горячая клавиша | Действие |
|---|---|---|
| **Предыдущий** | — | Предыдущий предмет в отфильтрованном списке |
| **Следующий** | — | Следующий предмет |
| **Следующий необработанный** | — | Перейти к ближайшему необработанному предмету |
| Сохранить прогресс | **Ctrl+S** | Явное сохранение |

После нажатия кнопок навигации фокус автоматически возвращается на главную таблицу, а выбранная строка прокручивается в видимую область.

## Браузер NPC

Секция **"NPC выбранного предмета"** в правой панели:

- **Список NPC** — все NPC, с которых падает выбранный предмет. В заголовке отображается количество в скобках. Клик по NPC обновляет нижнюю таблицу.
- **Таблица "Все предметы выбранного NPC"** — все предметы выбранного NPC с текущими группами/шансами и Loot Pack ID. В заголовке отображается количество строк в скобках.
  - **Одиночный клик** — фокус переходит на главную таблицу
  - **Двойной клик** — переход к этому предмету в главной таблице (фильтры сбрасываются автоматически)

## Сохранение и экспорт

| Действие | Результат |
|---|---|
| **Ctrl+S** / "Сохранить прогресс" | Сохраняет рабочий файл в папку `Data\` рядом с exe |
| Авто-сохранение | Происходит автоматически после каждого "Применить" / "Сбросить" |
| **"Экспорт результата"** | Сохраняет итоговый JSON: `{ "items": [ { "item_id": X, "chance": Y }, ... ] }` |
| **"Записать в БД"** | Записывает все назначенные значения напрямую в SQLite: upsert таблиц `loots` и `loot_groups`. Перед записью показывает превью (сколько строк будет обновлено/добавлено). |
| **"Импорт SQL в БД"** | Выполняет SQL-файл напрямую в базу данных. Можно выбрать несколько файлов сразу. Поддерживаются обычный SQL и дампы Navicat (INSERT без имён колонок). Новые строки добавляются, существующие (по `id`) обновляются без удаления. |
| **"Генерация патча из БД"** | Читает таблицы `loot_actability_groups`, `loot_groups`, `loot_pack_dropping_npcs`, `loots` из выбранной SQLite-базы и генерирует SQL-патч (`INSERT OR REPLACE`) в папку `Data\`. Показывает превью с количеством строк по таблицам и путь к файлу. БД **не изменяется** — патч применяется вручную через «Импорт SQL в БД». |

### Статистика импорта

Секция **"Сводка"** показывает суммарную статистику двух типов импорта рядом:

```
── Импорт SQL ──        ── Генерация патча ──
Добавлено:  N           Добавлено:  N
Обновлено:  M           Обновлено:  M
Итого:      N+M         Итого:      N+M
```

Статистика накапливается между сессиями и сохраняется в файле прогресса.

## Расположение файлов

Все данные хранятся **рядом с исполняемым файлом** в папке `Data\`:

```
LootMaster.exe
Data\
  loot_group_progress.json      ← рабочий файл прогресса (включает статистику импортов)
  column-settings.json          ← расположение и ширина столбцов, ширина панелей, размер окна, язык
  patch_YYYYMMDD_HHMMSS.sql     ← патч-файлы, сгенерированные через «Генерация патча из БД»
```

## Формат файлов

### Рабочий файл прогресса (`Data\loot_group_progress.json`)
```json
{
  "db_path": "C:/path/to/compact.sqlite3",
  "loot_db_path": "C:/path/to/compact.server.table.sqlite3",
  "source_json_paths": ["C:/path/to/loot1.json", "C:/path/to/loot_doodad.json"],
  "items": {
    "12345": { "group": 1, "chance": 0.5, "min_amount": 1, "max_amount": 3, "grade_id": 2, "always_drop": false }
  },
  "categories": {
    "100": { "group": 2, "chance": 1.0, "grade_id": 1, "always_drop": false }
  },
  "sql_imports": { "patch.sql": { "inserted": 100, "updated": 50 } },
  "db_imports":  { "source.sqlite3": { "inserted": 4109, "updated": 0 } }
}
```

### Входной JSON лута (NPC)
```json
[
  {
    "npc_id": 123,
    "items": [
      { "item_id": 456 },
      { "item_id": 789 }
    ]
  }
]
```

### Входной JSON лута (Doodad)
```json
[
  {
    "doodad_id": 322,
    "loot_pack_id": 6414,
    "items": [
      { "item_id": 456, "name": "Some item", "count": 1 }
    ]
  }

```

### Экспортный JSON
```json
{
  "items": [
    { "item_id": 456, "chance": 0.5 },
    { "item_id": 789, "chance": 1.0 }
  ]
}
```

## Типичный рабочий процесс

```
1. Открыть БД + JSON (один раз, далее восстанавливается автоматически)
2. Включить "Только необработанные"
3. По очереди обрабатывать предметы:
   - Если вся категория одинакова → "Применить ко всей категории"
   - Если предмет особенный → "Применить" только к нему
4. Используй браузер NPC чтобы видеть контекст дропа
   - Колонки "Группа (БД)" и "Шанс (БД)" показывают, что уже есть в базе
   - Loot Pack ID помогает понять, к какому пулу лута относится предмет
5. Ctrl+S периодически
6. Когда всё готово:
   - "Записать в БД" — записать напрямую в SQLite (upsert loots + loot_groups)
   - или "Экспорт результата" — сохранить JSON для дальнейшей обработки
   - или "Импорт SQL" — загрузить SQL-дамп напрямую в БД лута
   - или "Генерация патча из БД" — сгенерировать патч из другой SQLite-базы, затем применить через "Импорт SQL"
```
